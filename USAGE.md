### 功能说明

| 功能 | 操作 | 说明 |
|------|------|------|
| 浏览图片 | 滚动页面 | 瀑布流自动排列，懒加载 |
| 查看详情 | 点击图片 | 跳转到详情页 |
| 查看信息 | 悬停图片 | 显示标题、ComfyUI 标签、时间 |

| 上传作品 | 点击上传按钮 | 跳转到上传页面 |

### 瀑布流特性

- **自适应列数**：PC 4列 → 平板 3/2列 → 手机 1列
- **懒加载**：图片进入视口前不加载
- **入场动画**：图片依次淡入显示
- **悬停效果**：卡片上浮 + 发光边框

### 上传流程

#### 1. 选择图片

- 点击上传区域或直接拖拽图片
- 支持格式：PNG、JPG、WebP
- 建议图片大小不超过 10MB

#### 2. 填写信息

**必填项**：
- 正向提示词（Prompt）
- 模型名称（Model）

**选填项**：
- 作品标题
- 负向提示词
- 采样器、步数、CFG Scale
- 分辨率（宽/高）
- 种子数
- VAE、CLIP Skip
- ControlNet、IP-Adapter、LoRA
- 其他参数

#### 3. 上传

1. 点击 **上传作品** 按钮
2. 等待上传完成（图片 → 参数 → 索引）
3. 上传成功自动显示提示
4. 返回首页查看新作品

## ComfyUI 参数说明

画仓支持记录和展示完整的 ComfyUI 工作流参数。

### 参数分组

#### 核心参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 模型 (Model) | 基础模型文件 | `sd_xl_base_1.0.safetensors` |
| 采样器 (Sampler) | 采样算法 | `DPM++ 2M Karras`, `Euler a` |
| 调度器 (Scheduler) | 噪声调度方式 | `karras`, `normal`, `exponential` |
| 步数 (Steps) | 采样步数 | `20-50` |
| CFG Scale | 引导强度 | `7.0-12.0` |
| 种子 (Seed) | 随机种子，-1 表示随机 | `1234567890` 或 `-1` |

#### 图像设置

| 参数 | 说明 | 示例 |
|------|------|------|
| 宽度 (Width) | 输出图片宽度 | `512`, `768`, `1024` |
| 高度 (Height) | 输出图片高度 | `512`, `768`, `1024` |
| 批次大小 (Batch Size) | 每批生成数量 | `1-4` |
| 批次数量 (Batch Count) | 总批次数 | `1-10` |

#### 高级设置

| 参数 | 说明 | 示例 |
|------|------|------|
| VAE | 变分自编码器 | `sdxl_vae.safetensors` |
| CLIP Skip | 跳过 CLIP 层数 | `1`, `2` |
| 降噪强度 | 图生图去噪程度 | `0.5-0.8` |

#### 扩展模块

| 参数 | 说明 |
|------|------|
| ControlNet | 控制网络模型，用于姿态、边缘等控制 |
| IP-Adapter | IP-Adapter 模型，用于风格/内容参考 |
| LoRA | LoRA 模型和权重 |

### 参数文件格式

存储为 JSON 格式：

```json
{
  "prompt": "masterpiece, best quality, highly detailed, 8k...",
  "negativePrompt": "lowres, bad anatomy, bad hands, text...",
  "model": "sd_xl_base_1.0.safetensors",
  "sampler": "DPM++ 2M Karras",
  "scheduler": "karras",
  "steps": 30,
  "cfgScale": 7.0,
  "width": 1024,
  "height": 1024,
  "seed": 1234567890,
  "vae": "sdxl_vae.safetensors",
  "clipSkip": 2,
  "controlnet": "control_v11p_sd15_canny",
  "lora": "detail_enhancer:0.8\ncinematic:0.5"
}
```

## 常见问题

### 图片相关

**Q: 图片显示不出来？**

检查项：
1. R2 公开访问是否开启 → 参考 CONFIG.md
2. CORS 是否正确配置 → 参考 CONFIG.md
3. 图片 URL 是否正确 → 检查浏览器开发者工具 Network

**Q: 图片加载很慢？**

可能原因：
1. R2 Bucket 距离用户较远 → 使用 R2 自定义域名
2. 图片文件过大 → 建议压缩到 2-5MB
3. 网络连接问题 → 检查本地网络

**Q: 支持什么图片格式？**

支持：PNG、JPG、WebP

---

### 上传相关

**Q: 上传失败怎么办？**

检查项：
1. R2 API 密钥是否正确 → 重新生成令牌
2. CORS 是否包含 PUT/POST → 检查 R2 CORS 设置
3. 网络连接是否正常
4. 图片大小是否超限（建议 < 10MB）

**Q: 上传后首页不显示？**

上传成功后会更新 index.json，刷新首页即可看到新图片。

**Q: 可以批量上传吗？**

目前仅支持单张图片上传。

---

### 配置相关

**Q: 配置丢失了怎么办？**

访问 `upload.html`，重新进行 R2 配置。配置存储在浏览器 localStorage 中。

**Q: 如何修改 R2 配置？**

1. 访问 `upload.html`
2. 登录后进入配置表单
3. 修改配置并保存

**Q: 如何修改管理员密码？**

清除浏览器 localStorage 中的 `gallery-config`，然后重新设置。
